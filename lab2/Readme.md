LAB 2: Lexical Analyzer for Token Recognition Using Flex (Lex)

Objective
->To understand the concept of lexical analysis and implement a Flex (Lex) program to convert source code into tokens.
->To identify and classify different tokens such as keywords, identifiers, numbers, operators, and separators from input code.

Theory
Lexical Analysis is the first phase of a compiler. It reads the source code character by character and 
converts it into meaningful tokens called lexemes.A tool called Flex (Fast Lexical Analyzer Generator) 
is used to automatically generate lexical analyzers.
The lexical analyzer identifies the following token types:1Keywords
1)Identifiers
2)Numbers
3)Operators
4)Separators 
5)(Delimiters)


Requirements
macOS / Linux / Windows
Flex (Lex)
GCC Compiler




Result
The lexical analyzer was successfully implemented using Flex (Lex). The program correctly identified keywords, 
identifiers, numbers, operators, and separators from the input source code.


Conclusion
This lab helped in understanding the working principle of lexical analysis and the use of Flex for generating
lexical analyzers in compiler design. It also improved understanding of how source code is broken into tokens
during compilation.
